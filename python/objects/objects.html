What is OOP?

  • A way of looking at problems
  • Break problems into objects
  • Objects have properties and methods
  • (and sometimes events)
  • Objects encapsulate functions and data

Importing a module

  • Namespaces and Modules audio file

Importing and namespaces

  • useCritter.py imports critter.py
  • python3 useCritter.py
  • python3 critter.py
  • ...but there's a problem...
  • The import command runs any python code in the module
  • We want to run the class definition,
  • but not the main() function

Couldn't you eliminate main()?

  • One solution would be to eliminate main() in critter.py
  • Then critter.py would not be executable
  • It would only be for importing into other programs
  • However, it's nice to have a main() in library modules
  • It's a great place to test the functions or objects

Introducing namespaces

  • Python uses namespaces
  • A namespace is defined whenever you run a program
  • If you import a module, you also define a namespace
  • Run critter.py and look at the namespace (__name__)
  • The namespace is "__main__" for the currently running module

Namespaces and importing

  • Now run useCritter.py
  • Its own namespace is "__main__" (because it's the running program)
  • When it imports critter.py, look at critter's namespace
  • Critter now says its namespace is "critter" (the module name)

Running only when you want...

  • The __name__ variable can be used to see if code is run as a module
  • If __name__ is "__main__", the code has been directly run
  • If __name__ is something else, the code has been imported
  • Only run the main() function if
    __name__ == "__main__"

Building critter as a module

Access Methods

  • Access Modifiers audio file
  • accessMethods.py
  • python3
  • It's common to provide special methods to a class
  • a get method allows you to get the value of a property
  • a set method allows you to modify the value of a property
  • getters and setters are called access methods

Get Methods

  • A getter returns the value of a variable
  • It can filter or modify the output
  • Getters usually have only the self paramter
  • They return a property value
  • It's common to have a getter for each property

Set Methods

  • A setter is used to change the value of a property
  • It can filter and modify the input
  • Setters normally accept a property parameter
  • This value is often tested or modified, then passed to a property
  • It's common to have a setter for each property

Access methods protect data

  • Frequently you will want to enforce some rules about data
  • Imagine you want a circle class
  • However, the radius can't be a negative number,
  • ...and it can't be larger than 100
  • Also, you want to be able to get the diameter
  • You shouldn't be allowed to change the diameter,
  • ...because it's dependent on the radius
  • Access methods can enforce these rules

Case Study: the Circle class

  • Look at circle.py
  • python3
  • This class enforces the rules about radius
  • The setRadius method enforces radius rules
  • By the time the function is finished, radius has a legal value
  • Even the constructor uses setRadius

Access methods and virtual properties

  • Getters also help work with data
  • The getRadius() method is very straightforward
  • It simply returns the radius
  • GetDiameter() is special
  • There is no diameter property (nor should there be)
  • The diameter is derived from the radius when needed
  • The setDiameter() method actually changes the radius

Making 'private' properties

  • In circle.py, it's still possible to directly access radius
  • This can cause illegal values
  • It would be great if we could force users to use functions
  • That way, the data would always be protected
  • The answer is called data-hiding

The private circle

  • Look at circlePrivate.py
  • python3
  • This program is very much like circle.py
  • However, all references to self.radius are now self.__radius
  • The double underscore makes the property hidden
  • The name is quietly changed by Python so data can ONLY be accessed by methods
  • The only access to __radius is in the access methods
  • Even the constructor uses access methods

Building a virtual property

  • Data hiding is great, but it would be nice to have a radius property
  • Python has a special function that creates a virtual property
  • the property function allows you to assign get and set functions
  • Look at the main code starting at line 50
  • It looks to the programmer like we have a radius property again
  • There's also a diameter property!

The dirty secret of properties

  • Look at circleProperty.py
  • python3
  • When the user assigns a value to a property, the program passes data through the associated setter
  • Likewise, data is passed through the getter when the user retrieves a property
  • The end result is easy for the programmer, and it protects the data
  • You can even add a property for a virtual element like diameter
